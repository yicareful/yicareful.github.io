解决QQ.exe启动无响应的问题

发表于2026-02-05|更新于2026-02-06|软件问题
|总字数:770|阅读时长:2分钟

问题点

  由于新版本的QQNT不再检测QQ.exe同级目录下的dgbhelp.dll文件，如果不将QQ.exe替换成LiteloaderQQNT电报群组中的版本的话，直接安装对应的插件，无法绕过QQ的检测，会导致打开QQ提示“文件已损坏，请重新安装QQ”。如下图所示：

  替换之后，虽然这个报错提示是没了，但是又遇到了另一个问题: 双击QQ.exe之后没有任何反应，只能通过命令行的方式启动才能打开QQ，如图所示：

解决办法

  于是我灵机一动：既然现在桌面上的快捷方式是QQ.exe本身，但是想打开的话又只能通过命令行，那我直接创建一个脚本，让这个脚本通过命令行来运行QQ.exe，然后在桌面创建一个快捷方式指向这个脚本不就行了？我真是太天才了。接下来是具体步骤：

创建VBS启动脚本

  新建一个记事本，在记事本里输入以下这段代码：

  注意第4行的路径要填写实际QQ.exe的绝对路径，至于怎么查找这个路径：在桌面右键QQ的图标，然后点击打开文件所在的位置。
  点击文件-另存为：

  • 文件名: 启动QQ.vbs(必须以.vbs结尾，不是.txt)
  • 保存类型: 所有文件
  • 编码: ANSI(这里解决乱码问题，不改也行)
  • 保存位置: 选一个固定的文件夹比如D:\，最好不要直接放桌面。

创建桌面的快捷方式

  桌面空白处右键 -> 新建 -> 快捷方式，在请输入对象的位置里，点击浏览找到刚才保存的启动QQ.vbs，选中后点击下一步。然后将快捷方式重命名为QQ(什么名字都行，你喜欢就好)，点击完成。现在双击这个快捷方式就能一键启动QQ了，不用专门打开命令行。

快捷方式换个图标

  刚创建好的这个快捷方式图标非常丑，而且不看名字分辨不出来这是干啥的，这里把它换成QQ的图标就一目了然了。右键刚才创建好的快捷方式 -> 属性 -> 更改图标 -> 浏览，找到QQ的安装目录C:\Program Files\Tencent\QQNT\(这里以实际安装目录为准)，选中QQ.exe，选好后点确定保存。
